# ADR 028: Native Anytype properties everywhere ‚Äî the `gc_fields` blob retired

Date: 2026-07-12
Status: accepted

## Context

Since the space-reflecting pivot (ADR 006), story-node data has moved
steadily into real Anytype properties: summaries into the built-in
`description` (ADR 011), long-form descriptions into the object body
(ADR 010), story fields into native scalar properties with an explicit
minting opt-in (ADR 012/023), Scheduled Events onto minted, human-named
properties (ADR 027). One survivor remained: the `gc_fields` text
property ‚Äî a JSON blob invisible and uneditable in the Anytype UI ‚Äî with
two live uses:

1. **Session snapshots** (ADR 021): `AnytypeSessionStore` serialized each
   chat's session state into `gc_fields` on its `gc_session_context`
   node.
2. **Infra-role residual routing** (ADR 012): fields on intent/capture
   nodes with no matching property (`user_id`, `model`, `mode`,
   `generated_at`, `origin`) fell through into the blob.

Plus a read-side merge in `to_node` that folded legacy blob contents into
`Node.fields` for pre-ADR-023 story nodes.

A 2026-07-12 code audit confirmed the blob was load-bearing but also that
every use had a straightforward native-property replacement. With a fresh
project ahead (no existing worlds to migrate), the compat rationale that
ADR 012/023 cited for keeping it is gone.

## Decision

**Every server-written datum lives in a real Anytype property (or the
object body).** The ADR 027 pattern ‚Äî `gc_`-keyed properties minted at
bootstrap with human display names ‚Äî is the general rule for
infrastructure data, not a Scheduled-Events special case. Concretely:

- **Session snapshots** move to a dedicated `gc_chat_session` ("Chat
  session", text) property on the SessionContext node, minted at
  bootstrap. It is server-managed JSON, so it is deliberately NOT in
  `GC_REFLECTED_FIELD_KEYS` ‚Äî it never surfaces in `Node.fields`.
- **Attribution stamps** become five minted properties, keys owned by
  `domain/attribution.py` (the `domain/scheduling.py` pattern):
  `gc_generated_at` ("Generated at"), `gc_user_id` ("Generated for
  user"), `gc_model` ("Generated by model"), `gc_mode` ("Generated in
  mode"), `gc_origin` ("Origin message") ‚Äî all text, attached inline to
  `gc_intent` (all five) and `gc_prose` (timestamp only) at type mint.
  They join `GC_REFLECTED_FIELD_KEYS`, so recorder writes resolve like
  any native field and read back into `Node.fields` (the provenance sort
  in `NodeReader` consumes `gc_generated_at`). Display names are
  attribution-flavored on purpose: `field_property` matches display
  names on write, and bare "Model"/"Mode"/"Origin" are plausible user
  vocabulary.
- **The infra fall-through is deleted.** `_resolve_field_entries` has one
  native-only path for every role: an unmatched key raises
  `UnknownFieldKey` whether the node is a Character or an Intent. The
  in-memory fake mirrors this by seeding the attribution specs into any
  field catalog (the fake's stand-in for the bootstrap guarantee).
- **No legacy read paths.** The `to_node` blob merge is gone ‚Äî a stray
  `gc_fields` property on a pre-pivot object is just another unreflected
  `gc_` key. `LEGACY_TYPE_ROLES` (pre-pivot `gc_character`-style type
  compat), the retired `gc_description`/`gc_summary` constants, and the
  two one-shot migration scripts are deleted with it. There is no
  migration: this system now assumes spaces written by this system.
- **Catalog hygiene.** Reflected `gc_` keys belong to dedicated surfaces
  (the `schedule` tool, the recorders), so they are excluded from the
  `field_catalog` offered to the LLM as story-field vocabulary ‚Äî both
  the "(any type)" bucket in the adapter and the fake's flat catalog.

## Consequences

- Humans see and can filter/sort by intent provenance in Anytype (who
  asked, which model, when) ‚Äî previously locked inside JSON.
- One write rule for every role; the contract suite pins the same
  behavior on both backends (`test_infra_attribution_fields_resolve_natively`,
  `test_infra_unmatched_field_errors_like_any_other`).
- Recorders must only write bootstrap-minted keys. A new stamp means a
  new entry in `domain/attribution.py` (+ display name in
  `mapping.PROPERTY_DISPLAY_NAMES`) ‚Äî the mint rides `ensure_schema`.
- Spaces written by pre-028 builds lose access to old session snapshots
  and blob-held fields (they remain untouched on the objects; nothing
  reads them). Accepted: no such spaces need to survive.
- The rule for future features: **if the server writes it, it is a real
  property (or the body) with a human display name.** No new JSON
  side-channels.

## Supersedes / amends

- Supersedes ADR 012's infra residual routing ("the blob is the bot's
  channel for unmatched keys") ‚Äî deleted.
- Amends ADR 021: the session snapshot slot is `gc_chat_session`, not
  `gc_fields`.
- Amends ADR 023: the legacy read merge is removed; native-only now
  applies to every role, not just story roles.
